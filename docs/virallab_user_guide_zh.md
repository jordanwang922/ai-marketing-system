# ViralLab 中文使用指引

## 1. 系统是做什么的

`ViralLab / 爆款实验室` 是一个面向内容团队的爆款内容工作台。当前版本重点支持小红书图文内容，主流程是：

1. 输入关键词采集热门或最新内容
2. 查看采集到的真实样本
3. 对样本做 AI 分析
4. 提炼爆款 Pattern
5. 生成新的内容草稿

当前界面默认中文，登录后即可使用。

---

访问地址：

- 前端控制台：[http://127.0.0.1:3200/](http://127.0.0.1:3200/)

演示账号：

- 邮箱：`demo@virallab.local`
- 密码：`demo123456`

登录后，顶部可以看到：

- 总览
- 采集
- 样本
- 分析
- 模式库
- 生成

如果你想切换英文，可以在页面右上角切换 `中文 / EN`。

## 3. 第一次使用时应该怎么操作

建议第一次按下面顺序走：

1. 先配置小红书 Cookie
2. 验证 Cookie 是否有效
3. 发起采集任务
4. 在 `Collection Jobs` 看任务状态
5. 在 `Samples` 看采集成果
6. 再去看 `Analyses / Pattern Library / Latest Draft`

如果你跳过第 1 步和第 2 步，真实采集通常会失败。

---

## 4. 什么是 Cookie，为什么要配置

小红书的真实采集依赖你当前登录后浏览器里的身份状态。

这个登录状态会通过 Cookie 传给 ViralLab。  
没有 Cookie，或者 Cookie 已经过期，系统就无法稳定访问搜索结果和详情内容。

你可以把它理解成：

- Cookie 有效：系统知道“你已经登录过小红书”
- Cookie 失效：系统会被小红书当成未登录访客

当前自研采集主通道是：

- `xiaohongshu-playwright`

这条通道在真实采集前，会检查当前保存的 Cookie 状态。

---

## 5. 如何获取小红书 Cookie

### 方法一：通过浏览器开发者工具获取

推荐使用你平时登录小红书的 Chrome 浏览器。

操作步骤：

1. 在浏览器里打开小红书官网  
   建议先打开：[https://www.xiaohongshu.com/explore](https://www.xiaohongshu.com/explore)

2. 确认你已经是登录状态  
   如果没有登录，请先扫码登录

3. 打开开发者工具  
   Mac 常用快捷键：
   - `Option + Command + I`

4. 进入 `Application` 面板  
   然后在左侧找到：
   - `Storage`
   - `Cookies`
   - 选择 `https://www.xiaohongshu.com`

5. 找到 Cookie 列表  
   常见会看到很多键值，比如：
   - `a1`
   - `web_session`
   - `webId`
   - 其他小红书相关 cookie

6. 复制 Cookie  
   最简单的方式是到 `Network` 面板中任选一个发往 `xiaohongshu.com` 的请求，复制它的 `Cookie` 请求头整串内容

你最终需要的是这样一整串文本：

```text
a1=xxx; web_session=xxx; webId=xxx; ...;
```

不要只复制单个字段。  
请复制完整 Cookie 字符串。

### 方法二：重新扫码登录后由系统接管浏览器会话

如果你不想手工找 Cookie，也可以用我们联调时那种方式：

1. 先打开登录页
2. 人工扫码登录
3. 由系统从当前浏览器会话里抓取 Cookie

这个方式更适合开发联调，不适合普通用户每天自己操作。

---

## 6. 在系统里怎么保存 Cookie

登录 ViralLab 后，在页面里找到平台账号相关区域。

当前你要使用的是：

- 小红书平台账号

操作步骤：

1. 找到小红书 Cookie 输入区域
2. 把刚才复制的整串 Cookie 粘贴进去
3. 点击保存

保存之后，账号状态一般会先变成：

- `saved`

这表示：

- 系统已经收到了 Cookie
- 但还没有确认它是否真的还能用

---

## 7. 如何验证 Cookie 是否有效

保存 Cookie 后，建议立刻点击：

- `Verify Cookie`

系统会自动做这些事：

1. 启动浏览器
2. 带着你保存的 Cookie 打开小红书搜索页
3. 检查是否能正常访问搜索结果
4. 返回验证结果

验证后常见状态有三种：

### 1. `verified`

说明：

- Cookie 有效
- 可以进行真实采集

这是最理想状态。

### 2. `saved`

说明：

- 已保存
- 但还没手动验证

系统在某些情况下会在你第一次真实采集时自动触发验证，但仍建议你先手动点一次验证，这样更稳。

### 3. `invalid`

说明：

- Cookie 失效
- 当前不允许真实采集

这时系统通常会阻止任务进入真实采集流程。

---

## 8. Cookie 失效以后应该怎么处理

这是最常见问题之一。

### 常见失效表现

你会在 `Collection Jobs` 或 `Collector Debug` 里看到类似提示：

- `code=-101`
- `无登录信息，或登录信息为空`
- `login-required`
- `network-auth-required`
- `canCollect: false`

这表示：

- 当前保存的 Cookie 已经过期
- 或者当前登录状态已经不被小红书接受

### 正确处理方式

不要反复重试同一个失败任务。  
正确步骤是：

1. 回到浏览器重新打开小红书
2. 重新扫码登录
3. 重新复制最新 Cookie
4. 回到 ViralLab 覆盖保存旧 Cookie
5. 再点击 `Verify Cookie`
6. 看到状态变成 `verified` 后，再重新发起采集

### 不建议的做法

- 不要只刷新 ViralLab 页面
- 不要连续点很多次真实采集
- 不要继续使用旧 Cookie 反复重试

这样通常不会解决问题，只会不断产生失败任务。

## 9. 如何发起一次真实采集

在页面的 `采集` 区块中填写：

### 关键词

比如：

- `AI教育`
- `AI学习`
- `教育规划`

### 数量

建议首次先从：

- `5`
- `10`

开始

### 排序

当前常用有：

- `latest`：看最新内容
- `hot`：看热度内容

### Provider

当前主用请选择：

- `xiaohongshu-playwright`

然后点击创建采集任务。

## 10. 我应该在哪里看采集成果

这是最重要的问题。

### 第一处：`Collection Jobs`

这里是“任务看板”，主要用来确认任务有没有成功。

你会看到：

- 任务状态
- 关键词
- 采集数量
- provider
- 是否抓到样本
- 是否失败
- 失败原因
- 部分诊断信息

你应该先在这里确认：

- 状态是不是 `completed`

如果不是 `completed`，先不要急着往下看样本。

### 第二处：`Samples`

这里才是“采集成果”的主展示区。

每条样本会显示：

- 标题
- 作者
- 发布时间
- 正文片段
- 封面图
- 图片数量
- 视频数量
- 标签
- 点赞/评论/收藏/分享
- 质量分
- 原始来源链接

如果你想看“系统到底抓到了什么内容”，请主要看这个区域。

### 最推荐的判断顺序

1. 先看 `Collection Jobs`
2. 确认任务成功
3. 再看 `Samples`
4. 判断这批样本是不是你想要的内容

## 11. 样本质量怎么看

每条样本现在都有：

- `Quality xx/100`

这个分数不是热度分，而是“字段完整度分”。

它会综合判断：

- 有没有稳定内容 ID
- 有没有作者
- 有没有发布时间
- 正文是不是足够完整
- 有没有封面和媒体资源
- 链接是不是标准 note 链接
- 标签是否完整

一般来说：

- 分数越高，越适合进入后续 AI 分析
- 分数太低，说明样本字段不完整，需要继续优化采集

页面里还有：

- `Sample Quality`

这个面板会统计：

- 平均质量分
- 强样本数量
- 弱样本数量
- 最常见缺口

## 12. 采集完成后怎么继续用

采集只是第一步。

后面你可以继续看：

### `Analyses`

这里看每条样本的分析结果，比如：

- 钩子类型
- 结构类型
- 摘要
- 是真实 LLM 结果还是 fallback

### `Pattern Library`

这里看系统从多条样本里提炼出的爆款模式。

### `Latest Draft`

这里看基于 Pattern 生成的新内容草稿。

## 14. 常见问题

### 为什么我明明保存了 Cookie，还是不能采集

可能原因：

1. Cookie 只是 `saved`，还没验证
2. Cookie 已经过期
3. 小红书当前登录态已经被服务端判定失效

优先处理方法：

1. 点 `Verify Cookie`
2. 如果失败，重新扫码登录并重新保存 Cookie

### 为什么任务创建成功了，但没有样本

先去 `Collection Jobs` 看状态和原因。  
常见原因不是系统崩了，而是：

- 登录态失效
- 搜索结果为空
- 当前页面结构变化

### 为什么生成效果一般

先不要第一时间怀疑 AI。  
建议先看：

1. `Samples` 的质量分高不高
2. `Latest Real Workflow` 的 diagnostics
3. Pattern 和 Generate 是不是走了真实 LLM

很多时候，是上游样本质量不够，不是生成模型不行。

---

## 15. 推荐使用习惯

为了减少出错，建议你按这个习惯使用：

1. 每次开始前先确认 Cookie 是 `verified`
2. 先抓 `5~10` 条，不要一上来抓太多
3. 先看 `Collection Jobs`
4. 再看 `Samples`
5. 样本质量没问题，再跑 workflow
6. 如果 Cookie 失效，先重新登录，不要盲目重试

---

## 16. 当前版本的现实边界

当前版本已经可以做真实测试和内部使用，但还不是完全商用稳定版。

你要有这个预期：

- 小红书登录态会过期
- 平台页面结构未来可能变化
- 真实采集仍需要持续调优

但当前系统已经足够用于：

- 关键词调研
- 爆款样本收集
- AI 分析
- Pattern 提炼
- 内容草稿生成

---

## 17. 一句话总结

如果你只记住一句话，请记住这个顺序：

`先验证 Cookie -> 再发起采集 -> 先看 Collection Jobs -> 再看 Samples`

这就是当前 ViralLab 最正确、最省事的使用方法。
