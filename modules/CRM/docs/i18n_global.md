# 国际化与全球市场策略

语言策略
- 初期支持：中文 + 英文。
- 设计与文案需可扩展：所有文本必须国际化。
- 未来目标：global-to-global，多语言可插拔。

实现要求
- 前端使用 i18n 方案（next-intl）。
- 语言偏好为用户级配置。
- 数据库字段默认不做多语言，按原始语言存储。
- 后续可通过 AI 翻译生成目标语言版本。
- AI 输出必须标注语言与目标市场。

平台与市场情报
- 系统内需有“目标市场画像”与“平台偏好”模块。
- 以国家/地区为维度维护：主流平台、用户画像、内容偏好、最佳发布时间。
- 示例：韩国市场需覆盖 Naver 与其用户内容偏好。
