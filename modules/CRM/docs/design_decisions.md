# 关键设计决策（已确认）

用户与品牌
- 用户只属于一个品牌。
- 主管可见其团队所有客户与线索。
- 团队成员之间互不可见客户资料。
- 主管之间互不可见其他主管与团队资料。

线索分配
- 官网表单线索进入共享池。
- 支持自动分配与人工分配两种方式。
- 自动分配规则待定，开发到该阶段再讨论。

线索流转
- 状态流转：New → Contacted → Qualified → Converted / Lost。
- 线索升级为客户：
  - 轻咨询付费
  - 签约正式服务

AI 评估
- 评分范围：0-100。
- 触发方式：手工点击触发，不自动评估。
- 流转方式：人工判断，不自动流转。

轻咨询
- 分配方式：手动指派专家。
- 专家库：记录专家姓名、国家、背景、擅长、报价与服务范围。
- 专家报价：按服务类型固定价格（轻量/重度）。
- 交付方式：
  - 客户邮件 → 专家
  - 专家语音/文本 → 系统整理 → 客户邮件

国际化
- 初期中英双语 UI。
- 语言偏好为用户级设置。
- 数据库存原始语言，后续通过 AI 翻译生成目标语言。

移动端
- 电脑端完整功能。
- 手机端 H5 响应式，重点覆盖核心流程。
线索去重
- 以公司名称为主键进行去重。
- 支持合并同公司多条线索。

通知机制
- 站内信与邮件为主。

邮件模板与 SLA
- 先保持简单，后续再细化。

数据安全
- 暂不扩展复杂策略，后续再完善。
